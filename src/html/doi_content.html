API Documentation

GET /v1/doi/{doi}
Download a paper PDF using its DOI identifier. The DOI should be provided without any URL encoding. 
# Example: Download a Nature paper
curl "https://$domain/v1/doi/10.1038/nature12373" \
  -H "Accept: application/pdf" \
  --output paper.pdf

# Example: Download a Science paper
curl "https://$domain/v1/doi/10.1126/science.1259855" \
  --output science-paper.pdf

# Using with localhost
curl "http://localhost:8080/v1/doi/10.1038/nature12373" \
  --output paper.pdf
Response Codes
The API returns standard HTTP status codes to indicate success or failure. 
200 OK - Paper successfully retrieved
400 Bad Request - Invalid DOI format (must start with "10." and contain "/")
404 Not Found - Paper not found or unavailable
500 Internal Server Error - Server error while fetching paper
DOI Format
The DOI must be provided as a bare identifier without any prefix URLs. 
✓ Correct: /v1/doi/10.1038/nature12373
✗ Wrong: /v1/doi/https://doi.org/10.1038/nature12373
✗ Wrong: /v1/doi/doi:10.1038/nature12373
